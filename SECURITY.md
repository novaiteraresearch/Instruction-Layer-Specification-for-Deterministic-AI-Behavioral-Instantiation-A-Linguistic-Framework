# Security Policy

## Scope

This repository contains documentation and citation metadata only. It does not contain executable software, deployed services, APIs, or network-accessible components.

As such, traditional software vulnerability disclosures (e.g., CVEs, remote code execution, injection attacks) do not apply to this repository directly.

---

## Relevant Security Concerns

Security-relevant concerns that may apply to this repository include:

1. **Misuse of documented methods in downstream systems**
   If you believe the instruction-layer specification methods documented in the associated publication are being applied in a downstream system in a manner that creates safety risks, misaligns AI behavior in harmful ways, or violates the terms of the proprietary license, please report this concern.

2. **Unauthorized redistribution of proprietary content**
   If you observe unauthorized reproduction, distribution, or deployment of content from this repository or the underlying technical specification in violation of the LICENSE, please report this.

3. **Metadata integrity concerns**
   If you identify a discrepancy in citation metadata (DOI, ORCID, author name, timestamps) that may constitute misattribution or fraud, please report this.

---

To report a security concern related to this repository or the underlying research:

- Author: Ariel J. Furlow

Please provide a clear description of the concern, including:
- What you observed
- Where you observed it
- Why you believe it constitutes a security or integrity issue
- Any supporting evidence

All reports will be reviewed in good faith and in a timely manner.

## Out of Scope

The following are out of scope for this security policy:

- Requests to disclose proprietary implementation details
- Requests to change the license terms
- General academic disagreements with the methods described in the publication
